# Proposal: GGA Pre-commit Integration

## Intent

Integrate GGA as a pre-commit hook for skillsync/TUI to auto-review staged Go code against project coding standards using the opencode AI provider.

## Scope

### In Scope
- Create `.gga` config with `PROVIDER=opencode`, `MODEL=opencode/deepseek-v4-flash-free`, `RULES_FILE=".gga-rules.md"`
- Create `.gga-rules.md` with Go + Bubbletea + project-specific coding standards
- Verify hook works end-to-end on a staged change

### Out of Scope
- Modifying AGENTS.md (auto-generated by skill-sync)
- Changing skill-sync behavior
- Installing GGA globally (already done)
- CI/CD or GitHub Actions integration
- Non-Go file review patterns

## Capabilities

### New Capabilities
- `gga-precommit`: GGA-based pre-commit code review — rules, config, and hook verification steps

### Modified Capabilities
- None — this is a new capability, no existing specs change

## Approach

GGA needs two files: `.gga` (config) and `.gga-rules.md` (rules). The hook is already installed from exploration. We source rules from `docs/contributing.md`, existing codebase patterns, and standard Go/Bubbletea conventions. Use `RULES_FILE=".gga-rules.md"` to decouple from AGENTS.md.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `.gga` | New | GGA config — provider, model, rules file reference |
| `.gga-rules.md` | New | Coding standards for Go + Bubbletea + project patterns |
| `.git/hooks/pre-commit` | Already exists | Calls `gga run \|\| exit 1` — no change needed |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Rules file and AGENTS.md drift | Med | `.gga-rules.md` is standalone, not derived from AGENTS.md |
| Hook blocks valid commits | Low | `gga` outputs review comments but exit code is configurable |
| opencode provider unavailable | Low | GGA reports error, user can `git commit --no-verify` |

## Rollback Plan

Delete `.gga`, `.gga-rules.md`, and the pre-commit hook. Restore via git checkout if committed.

## Dependencies

- GGA v2.8.0 (already installed system-wide via Homebrew + gga.ps1)
- opencode.exe on PATH (`C:\Users\ismar\.bun\bin\opencode.exe`)

## Success Criteria

- [ ] `gga run` succeeds when invoked from project root on a staged Go file
- [ ] Review output references rules from `.gga-rules.md` (not AGENTS.md)
- [ ] Pre-commit hook fires on `git commit` and runs GGA review
- [ ] `docs/contributing.md` rules are reflected in `.gga-rules.md`
